# تقرير تحديث ALOKA Live — الإصدار 4.0

تاريخ البناء: 2026-07-21

هذا الإصدار تحديث إضافي للمشروع الحالي، وليس إعادة بناء. بقيت مصادر البيانات وواجهات API ومسارات تشغيل المباريات ولوحة التحكم الحالية كما هي.

## ضمانات التوافق

- لم تُعدّل ملفات `AppChannels.php` و`AppLinks.php` و`ChannelLib.php` و`Streams.php`.
- بقي اكتشاف التطبيق بواسطة `User-Agent: com.aloka.live.app` كما هو حرفياً.
- بقي رابط التطبيق الأصلي بصيغة `intent://...#Intent;scheme=xmtv;package=com.aloka.live.app;end;` كما هو.
- لم تُحذف أي Route أو Controller أو View أو إعداد موجود.
- لم تتغير Endpoints الخاصة بالمباريات والأخبار أو TMDB؛ إدارة الأخبار الجديدة طبقة محلية اختيارية فوق نتيجة المصدر.
- كتالوج القنوات الجديد مستقل عن مكتبات قنوات وروابط المباريات القديمة.

## الإضافات

### القنوات

- صفحة `/channels` باللغتين العربية والإنجليزية، وتصنيفات ومجموعات وقنوات متجاوبة.
- لوحة إدارة جديدة: `/admin/channel-catalog` لإدارة التصنيفات والمجموعات والقنوات والترتيب والظهور والصور.
- أنواع تشغيل `intent` و`hls` و`external`، مع الجودة والحالة ونوع الوصول والمفضلة.
- حفظ الرابط أو الكود المشفر كما يُدخل، ثم تغليفه في نوع `intent` بالحزمة الرسمية الثابتة.
- نافذة Glass/Dark عند تعذر فتح تطبيق التشغيل، مع رابط `https://t.me/alokalive`.
- إدراج صفحات القنوات في Sitemap وبيانات `ItemList` المنظمة.

### الأخبار

- إظهار الكاتب والتصنيف في البطاقة وصفحة الخبر وبيانات Article المنظمة.
- اسم كاتب افتراضي قابل للتعديل من لوحة التحكم.
- لوحة إدارة جديدة: `/admin/editorial-news` لإضافة وتعديل وحذف وإخفاء الأخبار المحلية.
- دعم المسودة والمنشور والمخفي، وتأكيد قبل الحذف، ورفع الصور وتعديل وقت النشر.
- تعديل أو إخفاء أخبار API بطبقة محلية غير مدمرة؛ طريقة الجلب وEndpoint الأصليان لم يتغيرا.

### المسلسلات والمباريات

- بطاقات حلقات بصور TMDB وعنوان ومدة وزر تشغيل وحالة مشاهدة محلية.
- fallback لصورة المسلسل ثم placeholder عند غياب صورة الحلقة.
- تبويبات المواسم تستمر كروابط Server-rendered، ولا تتغير طريقة تشغيل الحلقة الحالية.
- أسماء الفرق في بطاقات المباريات و`Live Right Now` أصبحت تدعم سطرين على الجوال مع حجم خط مرن والاسم الكامل في `title`.

### الأداء والوصول

- صور القنوات والحلقات تستخدم `loading="lazy"` و`decoding="async"` مع أبعاد ثابتة.
- CSS وJavaScript أعيد بناؤهما إلى ملفات `.min` بواسطة esbuild.
- التصميم يدعم RTL/LTR والوضعين الداكن والفاتح ومقاسات اللمس والشاشات الصغيرة.
- نافذة التطبيق تدعم Escape وإغلاق الخلفية وحجز التركيز للوحة المفاتيح.
- تم تحديث Build token إلى `2026-07-21-v4.0` لتحديث أصول Service Worker دون طلب مسح الكاش يدوياً.

## ملفات الإعداد الجديدة

- `storage/settings/channel_catalog.json`
- `storage/settings/editorial_news.json`

يبدأ كتالوج القنوات فارغاً عمداً. تُضاف البيانات من لوحة التحكم، ولا توجد قنوات ثابتة داخل الكود.

## التحقق قبل التسليم

- فحص بنية 152 ملف PHP بواسطة parser مستقل.
- فحص JavaScript المصدر والمصغّر.
- التحقق من صحة ملفات JSON وSVG.
- مقارنة SHA-256 لملفات تشغيل القنوات والمباريات القديمة مع النسخة الأصلية.
- مقارنة دالتي `is_tofix_app()` و`app_intent_url()` حرفياً مع النسخة الأصلية.
- إعادة بناء ملفات CSS/JS المصغرة وفحص سلامة أرشيف ZIP النهائي.

ملاحظة: يجب تشغيل فحص Lighthouse النهائي على دومين الإنتاج بعد النشر، لأن نتيجته تتأثر بزمن الخادم وواجهات الطرف الثالث والشبكة وبيانات الصفحة الحقيقية.
