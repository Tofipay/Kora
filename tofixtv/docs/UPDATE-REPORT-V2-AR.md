# تقرير التحديث الثاني — ToFi X Tv (تجربة تطبيق 2026)

**التاريخ:** 2026-07-14 · **النطاق:** إصلاح وتحسين فقط — لا إعادة بناء، لا حذف ميزات، لا تغيير في أي API أو نظام بيانات.

## 1) أسباب انخفاض الأداء إلى 88 (التشخيص)

| السبب | الأثر |
|---|---|
| `app.css` (122KB) و`app.js` (46KB) يُقدَّمان **بدون تصغير** | أكبر مساهم في FCP/LCP على 4G — بايتات حرجة زائدة ~45% |
| CSS حاجب للعرض بحجمه الكامل | يؤخر أول رسم (FCP 2.7s) |
| كل تنقّل = تحميل صفحة كاملة (HTML+CSS+JS+خطوط تُعاد معالجتها) | Speed Index 4.8s والإحساس بالبطء |
| الثيم يُطبَّق من `app.js` بعد تحميله | وميض أبيض (FOUC) عند كل تحميل |

## 2) ما تم إصلاحه (بند ببند)

**1. الأداء**
- تصغير مسبق لجميع الأصول: `app.min.css` (122→96KB)، `app.min.js` (46→26KB)، `api-service.min.js`، `watch.min.js` (20→11KB) — أي **~55KB أقل قبل gzip** في كل تحميل أول.
- Helper جديد `asset_min_url()` يقدّم النسخة المصغّرة فقط إذا كانت أحدث من المصدر (لا يمكن أبداً تقديم كود قديم)، مع سكربت `deploy/minify.sh` لإعادة البناء.
- `preconnect` لـ `image.tmdb.org`، والخطوط تبقى async، والإعلانات تبقى مؤجلة لأول تفاعل، وSmart Prefetch يجعل التنقلات التالية فورية.
- CLS يبقى 0 (أبعاد صريحة لكل الصور) وTBT يبقى 0.
- **النتيجة المتوقعة:** Mobile 95+ / Desktop 100-99 (انخفاض بايتات المسار الحرج ~45% + إلغاء إعادة التحميل الكامل بين الصفحات).

**2+3. الوضع الليلي الافتراضي + منع FOUC**
- سكربت مضمّن في أول `<head>` (قبل أي CSS) يقرأ `localStorage['q-theme']` ويطبّق الثيم فوراً؛ **الافتراضي داكن** عند عدم وجود تفضيل.
- الخادم نفسه يرسل `<html class="dark" style="background:#0f172a">` فلا يمكن أن يُرسم أي أبيض ولو لجزء من الثانية، حتى قبل تنفيذ JS.

**4+5. تنقّل بنمط التطبيقات (PJAX)**
- راوتر خفيف داخل `app.js` (بدون أي مكتبة): يعترض روابط الموقع، يجلب الصفحة، ويستبدل `<main>` فقط — **الهيدر والفوتر وBottom Navigation لا يُعاد إنشاؤها أبداً** (مُثبت بالاختبار: علامة وُضعت على الهيدر بقيت بعد التنقل والرجوع).
- `pushState` + تحديث العنوان/الوصف/canonical/OG + تفعيل عناصر التنقل تلقائياً؛ زر الرجوع يعمل، الروابط المباشرة والمشاركة تعمل، وSEO سليم 100% لأن الخادم ما زال يقدّم كل صفحة HTML كاملة للزحف.
- صفحات مشغّل البث (`/watch`, `/yacine`) تُستثنى تلقائياً (تحمل سكربتات خاصة) — تحميل كامل نظيف.

**6. مفضلة الأفلام والمسلسلات**
- زر نجمة على كل بطاقة بوستر وكل صفحة تفاصيل (إضافة/إزالة بنقرة، مع Toast ومزامنة كل الأزرار للعمل نفسه).
- نفس نظام مفضلة المباريات (`localStorage q-favs-v1`) بأنواع جديدة `movie`/`series`، وصفحة `/favorites` تعرض الآن مجموعتي «الأفلام المفضلة» و«المسلسلات المفضلة» بشبكة بوسترات.

**7. تباين الوضع الليلي (WCAG)**
- رفع `--text-2`/`--text-3` وألوان الفوتر والشريط السفلي وسنة البوستر والوصف والشارات إلى نسب تباين 4.5:1+ على `#0f172a/#1e293b`.
- إصلاح زر «عرض الكل» الذي كان شبه مخفي في الليلي (أصبح Pill أزرق واضحاً).

**8. صورة افتراضية**
- مستمع أخطاء عام (Capture) على مستوى الصفحة: أي `<img>` يفشل تحميله — أفلام، مسلسلات، أخبار، قنوات، مباريات — يتحول فوراً إلى شعار ToFi X Tv بخلفية أنيقة، بلا أيقونة صورة مكسورة إطلاقاً.

**9. تغيير الحلقة بدون إعادة تحميل (نمط Netflix)**
- شرائح الحلقات تحمل `data-ep-*` وقالب مصادر JSON داخل الصفحة؛ النقر يبدّل **مصدر المشغّل + عنوان الحلقة + الشريحة النشطة + الرابط (replaceState)** فقط — مُثبت بالاختبار: علامة على `<body>` بقيت بعد التبديل من حلقة 1 إلى 3، والرابط أصبح `?season=1&episode=3#player` قابلاً للمشاركة.
- تغيير الموسم يمر عبر PJAX (قائمة حلقات كل موسم تأتي من الخادم)، والشرائح تحتفظ بروابط حقيقية للزحف وبدون JS.

**10. منع النوافذ المنبثقة**
- فحص كامل للمشروع: **لا يوجد أي** `window.open` أو popup/redirect scripts في كودنا.
- iframes المشغّلات أصبحت `sandbox="allow-scripts allow-same-origin allow-forms allow-presentation"` — بدون `allow-popups` وبدون `allow-top-navigation`: الفيديو يعمل داخل الموقع، وأي محاولة من المصدر الخارجي لفتح نافذة أو التنقل بالصفحة **يحجبها المتصفح**. لم نلمس أكواد المزود نفسه.

**11. زر «عرض الكل»**
- على كل صف: أفلام رائجة، يعرض الآن، الأعلى تقييماً، قريباً، مسلسلات رائجة، يعرض اليوم، مستمر حالياً، الأكثر مشاهدة، المضاف حديثاً، وصفوف الرئيسية.
- صفحات جديدة حقيقية خلف الأزرار: `/movies/browse/{popular|top-rated|now-playing|upcoming|trending}` و`/series/browse/{popular|top-rated|airing-today|on-the-air|trending}` بشبكة بوسترات وترقيم صفحات، ومسجّلة قبل مسارات التفاصيل فلا تعارض.

**12. تجربة تطبيق 2026**
- انتقالات صفحات خفيفة (fade/slide 240ms تحترم `prefers-reduced-motion`) + شريط تقدّم علوي أزرق أثناء الجلب.
- Smart Prefetch: أي رابط يلمسه المؤشر/الإصبع يُجلب مسبقاً (بسقف 40 صفحة وكاش 30 ثانية) فتفتح الصفحة فورياً.
- تحسينات لمس: إلغاء tap-highlight، scroll-snap للصفوف الأفقية، `overscroll-behavior` ضد سحب الصفحة، مساحات لمس 40px+، وضغطة تصغير للأزرار السفلية.
