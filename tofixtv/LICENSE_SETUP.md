# نظام التفعيل والترخيص — ALOKA Live

نظام تفعيل مركزي: تتحكم أنت (المطوّر/البائع) بتشغيل أي نسخة عن بُعد من صفحة Blogspot،
دون الحاجة للوصول لاستضافة العميل.

---

## 1) كيف يعمل باختصار

- عند تثبيت الموقع لأول مرة، **الموقع مقفل** ويطلب كود تفعيل داخل لوحة الإدارة.
- المالك يسجّل الدخول للوحة الإدارة → تظهر له **شاشة التفعيل** فقط → يُدخل الكود **مرة واحدة**.
- يتحقق الخادم (PHP) من الكود + الدومين الحالي + الحالة `status` من صفحة Blogspot.
- إذا كان الكود مطابقًا والدومين مصرّحًا و`status = active` → يُحفظ التفعيل، وتختفي شاشة التفعيل
  نهائيًا، ويعمل الموقع وكل الصفحات وواجهات API بشكل طبيعي.
- بعدها يتم **تحقق خلفي كل 10 دقائق** فقط (وكل دقيقتين أثناء الإيقاف لتسريع العودة)،
  وبين كل تحقق وآخر يُستخدم آخر نتيجة محفوظة محليًا — فلا يوجد طلب خارجي مع كل زيارة ولا يبطئ الموقع.
- إذا حُذف الترخيص، أو صار `status = disabled`، أو لم يعد الدومين مطابقًا →
  **يتوقف الموقع بالكامل** (الرئيسية + لوحة الإدارة + المشغّل + واجهات API) وتظهر صفحة احترافية فقط:
  «تم إيقاف تفعيل الموقع — للتواصل وإعادة التفعيل: t.me/Abu5Turkish» مع زر يفتح
  `https://t.me/Abu5Turkish`.
- عند إعادة `status` إلى `active` يعود الموقع للعمل **تلقائيًا** بعد التحقق التالي (خلال دقيقتين).

**الأمان:** كود التفعيل يُدخل عبر لوحة الإدارة ويُتحقق منه في PHP فقط، ويُحفظ من جهة الخادم في
`storage/settings/license.json` (خارج جذر الويب ومحمي بـ `storage/.htaccess`: `Require all denied`).
لا يُوضع الكود داخل JavaScript، ولا يُعرض بعد التفعيل، ولا يُحفظ في المتصفح.

### من يتحكم بالإيقاف/التشغيل؟ (مهم)

**التحكم من صفحة Blogspot فقط — وليس من ملف `license.json`.** ملف `license.json` الموجود على
استضافة العميل هو مجرّد **كاش مؤقّت** لآخر نتيجة (لتسريع الموقع)، وليس مصدر القرار. في كل تحقق
دوري يُعاد جلب الحالة من Blogspot ويُكتب فوق الكاش. فإذا وضعت `disabled` في Blogspot، يُقفل موقع
العميل في التحقق التالي مهما كان محتوى `license.json`.

وحُصِّن النظام ضد تلاعب العميل بالكاش المحلي:

- أي تاريخ **مستقبلي** في الكاش (محاولة لتأجيل الفحص للأبد) يُرفض ويُعامَل كأنه غير موجود.
- يجب أن يأتي **تأكيد فعلي من Blogspot** بأن الحالة `active` مرة واحدة على الأقل كل **12 ساعة**
  (نافذة `GRACE`)، وإلا يُقفل الموقع. لذلك لا ينفع أن يحرّر العميل الملف يدويًا ليُبقيه شغّالًا،
  ولا أن يحجب الوصول إلى Blogspot بجداره الناري ليمنع وصول أمر الإيقاف — سيُقفل الموقع خلال 12 ساعة كحدٍّ أقصى.
- **حذف** الملف لا يفيد العميل: يعود الموقع إلى حالة «غير مُفعّل» = مقفل (يحتاج إدخال كود صحيح،
  والكود لا يُقبل إلا إذا كان Blogspot يقول `active`).

> ملاحظة صريحة: إذا كان لدى العميل صلاحية كاملة على ملفات الاستضافة (كأي سكربت PHP يُباع)، فيمكنه
> نظريًا تعديل كود PHP نفسه لتعطيل الفحص — وهذا قيد يشمل جميع أنظمة الحماية المكتوبة بـ PHP على
> سيرفر العميل. لكن ضمن الاستخدام الطبيعي: **التشغيل والإيقاف بيدك أنت من Blogspot**، ولا يستطيع
> العميل إبقاء الموقع شغّالًا بتعديل `license.json` بعد أن توقفه.

## 2) الكود الذي تضعه داخل صفحة Blogspot

الصفحة: `https://admin-cod.blogspot.com/p/admin-cod.html`

من محرّر Blogspot، افتح وضع **HTML** (وليس Compose) والصق هذا الكود كما هو:

```html
<script id="aloka-license-data" type="application/json">
{
  "licenses": [
    {
      "code": "4DGH7DV6SHSS7SS",
      "domain": "example.com",
      "status": "active"
    }
  ]
}
</script>
```

الموقع يقرأ **فقط** محتوى العنصر `aloka-license-data`، ويحوّله إلى JSON، ثم يتحقق من:
`code` و `domain` و `status`.

> ملاحظة: النظام يتحمّل تلقائيًا أن يحوّل Blogspot علامات الاقتباس إلى `&quot;` أو يضيف `<br>`،
> فلا داعي للقلق من ذلك. لكن يُفضّل وضعه في صفحة/Page مستقلة كما في الرابط أعلاه.

## 3) إدارة العملاء

**لتفعيل عميل جديد** أضِف سجلًا جديدًا داخل `licenses` (افصل بين السجلات بفاصلة `,`):

```json
{
  "code": "كود العميل",
  "domain": "دومين العميل",
  "status": "active"
}
```

- `domain`: دومين العميل كما يظهر في المتصفح **بدون** `https://` و**بدون** `www`
  (مثال: `aloka-code.shop`). النطاقات الفرعية للدومين نفسه مسموح بها تلقائيًا (مثل `m.aloka-code.shop`).
- `code`: أي كود تختاره لهذا العميل (يُفضّل أن يكون طويلًا وفريدًا).

**لإيقاف موقع** غيّر حالته:

```json
"status": "disabled"
```

**لإيقاف نهائي** احذف سجل الترخيص من القائمة. في الحالتين يتوقف الموقع ويظهر صفحة الإيقاف.

**لإعادة التشغيل** أعِد `status` إلى `active` (أو أعِد السجل) — يعود الموقع تلقائيًا خلال دقيقتين.

مثال بأكثر من عميل:

```json
{
  "licenses": [
    { "code": "4DGH7DV6SHSS7SS", "domain": "aloka-code.shop", "status": "active" },
    { "code": "CLIENT-2-CODE",   "domain": "client2.com",     "status": "active" },
    { "code": "CLIENT-3-CODE",   "domain": "client3.net",     "status": "disabled" }
  ]
}
```

## 4) ملفات النظام (للمرجع)

| الملف | الدور |
|---|---|
| `app/Core/License.php` | كل منطق التحقق والكاش والقفل وصفحة الإيقاف. |
| `app/Views/admin/activate.php` | شاشة إدخال كود التفعيل داخل لوحة الإدارة. |
| `app/bootstrap.php` | بوابة الترخيص لكل الصفحات وواجهات API (عدا لوحة الإدارة التي تدير التفعيل بنفسها). |
| `public/api/_boot.php` | بوابة الترخيص لملفات `/api/*.php` المباشرة. |
| `app/Controllers/Admin.php` | القفل الصارم للوحة + مسار `admin/activate`. |
| `storage/settings/license.json` | حالة التفعيل المحفوظة محليًا (يُنشأ بعد التفعيل، محمي، لا يُرفع مع النسخة). |

**تغيير مصدر الترخيص (اختياري):** يمكنك تعيين متغيّر بيئة `ALOKA_LICENSE_URL` ليشير إلى صفحة أخرى
تحتوي نفس العنصر `aloka-license-data`؛ وإن لم يُضبط، يُستخدم رابط Blogspot أعلاه افتراضيًا.

**الإعدادات القابلة للضبط** داخل `app/Core/License.php`: مدة التحقق (`INTERVAL_ACTIVE` = 600ث،
`INTERVAL_LOCKED` = 120ث)، ونافذة السماح القصوى دون تأكيد من Blogspot (`GRACE_SECONDS` = 12 ساعة).
تصغير `GRACE_SECONDS` يزيد إحكام التحكم (يُقفل الموقع أسرع إن تعذّر الوصول إلى Blogspot)، لكنه قد
يوقف موقعًا سليمًا إذا انقطع Blogspot لمدة أطول من النافذة.

## 5) ملاحظة مهمة حول «حذف الملفات»

طلبت أن يقوم النظام بحذف/تخريب ملفات الاستضافة عند الإيقاف. **لم يتم تنفيذ هذا الجزء عمدًا**
لأنه مدمّر وغير قابل للتراجع: خطأ بسيط في JSON، أو انقطاع مؤقت للإنترنت، أو تغيير خاطئ للحالة،
قد يمسح موقع العميل نهائيًا دون إمكانية استرجاع. وقد استُبدل بـ **قفل كامل من جهة الخادم** يحقّق
نفس الهدف تمامًا (يوقف الموقع ولوحة الإدارة والمشغّل وواجهات API ويعرض صفحة الإيقاف فقط)،
مع ميزة حاسمة: **يعود الموقع للعمل فورًا** عند إعادة التفعيل — وهو ما يستحيل مع الحذف.
