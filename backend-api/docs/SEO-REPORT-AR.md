# تقرير إعادة هيكلة SEO — قمهد لايف (www.qamhad.com)

**التاريخ:** 2026-07-07
**الهدف:** المنافسة على الكلمات المفتاحية الرياضية عالية المنافسة في Google 2026، وعلى رأسها: **«مباريات اليوم بث مباشر»**
**الدومين الرئيسي الجديد:** `https://www.qamhad.com` — مع بقاء `live.qamhad.com` يعمل ويحوّل بالكامل.

---

## أولاً: ما تم إصلاحه

### 1. ترحيل الدومين (live → www) بدون فقدان أرشفة
| قبل | بعد |
|---|---|
| `SITE_URL` يتبع المضيف الذي يخدم الطلب (أي canonical متذبذب) | `SITE_URL = https://www.qamhad.com` ثابت — كل Canonical وhreflang وخرائط الموقع وJSON-LD تشير إلى www مهما كان المضيف (`app/config.php`) |
| `.htaccess` كان **يحذف** www (يحوّل www → بدون www!) | فرض `https://www.qamhad.com` بقفزة 301 واحدة من أي مضيف/بروتوكول (`public/.htaccess`) |
| لا يوجد تحويل على مستوى PHP | حارس 301 في `app/bootstrap.php` يعمل على أي استضافة حتى لو تعطل mod_rewrite |
| `robots.txt` يشير إلى خرائط live | يشير إلى خرائط www + خريطة المباريات الجديدة |
| nginx.conf باسم live فقط | Server blocks كاملة: أي مضيف قديم (qamhad.com / live / www.live) → 301 إلى www مع حفظ المسار |

### 2. أخطاء Google Search Console لبيانات SportsEvent
كل الحقول التي ظهرت في التقرير (انظر لقطة الشاشة) أصبحت تُولَّد تلقائياً في كل صفحة مباراة:

| الحقل | الحالة | المصدر |
|---|---|---|
| `startDate` | ✅ إلزامي — موجود دائماً | timestamp المباراة من JSON |
| `endDate` | ✅ | البداية + ساعتان عند غياب نهاية صريحة |
| `location` | ✅ | الملعب (Place) أو VirtualLocation لصفحة البث |
| `organizer` | ✅ | البطولة كـ SportsOrganization مع رابطها |
| `performer` | ✅ | الفريقان (SportsTeam مع الشعارات) |
| `offers` | ✅ | عرض مشاهدة مجاني (price 0) مع validFrom |
| `image` | ✅ | صورة OG بمقاس 1200×630 + شعارات الفريقين والبطولة |
| `eventStatus` | ✅ | EventScheduled |
| `description` | ✅ | وصف ديناميكي حسب حالة المباراة (قبل/أثناء/بعد) |

تم التحقق عملياً: صفحة مباراة تجريبية أخرجت SportsEvent كاملاً بكل الحقول التسعة، بلا أي حقل مفقود — جاهزة لاختبار Rich Results.

### 3. ترميز الروابط العربية
كل `hreflang` في `<head>` أصبح يمر عبر `absolute_url()` فيخرج مُرمّزاً بالنسبة المئوية (ASCII-safe) — كان يُطبع خاماً وقد يتشوه عبر البروكسيات.

## ثانياً: ما تمت إضافته

### 1. خرائط الموقع (Sitemaps)
- **`/sitemap.xml`** أصبح **فهرس خرائط** (Sitemap Index) يشير إلى الخرائط الخمس مع `lastmod`.
- **`/sitemap-match.xml`** — جديدة كلياً، تُولَّد ديناميكياً من JSON عند كل طلب وتغطي:

| النافذة | priority | changefreq | lastmod |
|---|---|---|---|
| مباريات **مباشرة الآن** | 1.0 | always | الآن |
| منتهية خلال 48 ساعة | 0.8 | hourly | وقت نهاية المباراة |
| **اليوم** (لم تبدأ) | 0.9 | hourly | تاريخ اليوم |
| **الغد** | 0.8 | daily | تاريخ اليوم |
| **بعد الغد** | 0.7 | daily | تاريخ اليوم |
| القادمة (+3 إلى +6 أيام) | 0.6 | daily | تاريخ اليوم |

  كل مباراة تظهر بالرابطين العربي والإنجليزي مع `xhtml:link hreflang` (ar / en / x-default)، وعنوان صفحتها يتولد بنمط: **«مباراة أمريكا وبلجيكا بث مباشر | قمهد لايف»** (تم التحقق منه فعلياً).
- `/sitemap-ar.xml` و`/sitemap-en.xml`: أضيف `lastmod` لكل رابط.
- `/sitemap-news.xml` (Google News) و`/sitemap-images.xml`: صور الأخبار في خريطة الصور أصبحت بمقاس **1200px** (متطلب Discover).

### 2. نظام الأرشفة الفورية (Indexing)
- **`app/Core/Ping.php`** + مسار **`/cron/ping`**: يراقب كل مباراة في نافذة (أمس ← غداً)؛ عند تغيّر حالة أو نتيجة أي مباراة (انطلاق، هدف، نهاية):
  1. يرسل روابط المباراة (ar+en) + الرئيسية + `/matches` + `/live` إلى **IndexNow** (Bing/Yandex/Seznam/Naver).
  2. يرسل **Ping** بخريطة `sitemap-match.xml` إلى Google وBing (مع مهلة 5 دقائق بين النداءات منعاً للإغراق).
  - الحالة محفوظة في `storage/settings/pingstate.json` — لا إرسال إلا عند تغيّر حقيقي.
  - أضِف إلى كرون الاستضافة: `* * * * * curl -s https://www.qamhad.com/cron/ping >/dev/null`
- **`Last-Modified` + `ETag` + `304 Not Modified`** (دالة `http_cache_validate()` الجديدة):
  - صفحات المباريات: ETag يتغير مع الحالة/النتيجة؛ مباراة منتهية تعيد **304** للزاحف (تم التحقق)، ومباراة مباشرة تعيد محتوى جديداً دائماً — توفير ضخم في ميزانية الزحف.
  - كل خرائط الموقع تدعم إعادة التحقق 304 أيضاً.

### 3. صفحات المباريات (Match SEO)
كل صفحة مباراة تحمل تلقائياً (كان معظمها موجوداً وتم استكماله):
- Title + Meta Description ديناميكيين حسب الحالة (موعد/بث مباشر/نتيجة)
- Canonical إلى www + Open Graph + Twitter Cards
- BreadcrumbList + SportsEvent + FAQPage (حتى 5 أسئلة: الموعد، القنوات، المعلق، النتيجة، الملعب)
- **جديد:** ItemList لمباريات اليوم (SportsEvent مكتملة العناصر) على صفحة كل مباراة وعلى صفحات القوائم `/matches` و`/live` — إشارة تغطية مباشرة لعبارة «مباريات اليوم بث مباشر».

### 4. المقال التلقائي أسفل كل مباراة
محرك `match_article()` (مُحقق عملياً: **327 كلمة، 7 عناوين H2** لمباراة مباشرة؛ يتراوح 300–800 كلمة حسب توفر البيانات) يتكيف مع الحالة:
- **قبل المباراة:** الموعد + القنوات الناقلة + التشكيلة المتوقعة + أهمية المواجهة.
- **أثناءها:** بث مباشر + متابعة لحظة بلحظة + النتيجة الحالية.
- **بعدها:** النتيجة النهائية + الملخص والأهداف + مقارنة الفريقين.

### 5. Google Discover
- وسم عام في `<head>` لكل الصفحات: `max-image-preview:large, max-snippet:-1, max-video-preview:-1`.
- صور المقالات الإخبارية أصبحت **1200px** في OG وSchema وخريطة الصور.
- `NewsArticle` + `Article` معلنان معاً في `@type`، مع `dateModified` و`inLanguage` و`isAccessibleForFree`.

### 6. تعدد اللغات
hreflang ar / en / x-default موجود في كل صفحة HTML **وفي كل خرائط الموقع** (x-default = العربية)، بروابط مطلقة على www ومرمّزة بأمان.

### 7. حزمة live.qamhad.com المنفصلة
مجلد/ZIP مستقل من 4 ملفات فقط (`.htaccess`, `index.php`, `robots.txt`, `README-AR.md`):
- 301 دائمة لكل مسار مع حفظ المسار والـ query string (قفزة واحدة).
- تحويل خرائط الموقع القديمة إلى الجديدة.
- robots.txt يحوّل إلى www (مع نسخة احتياطية تُبقي الزحف مفتوحاً — حجب الزحف يمنع انتقال الإشارات).
- شبكة أمان PHP لو تعطل mod_rewrite.

## ثالثاً: نقاط SEO المتبقية (يدوية — خارج الكود)

1. **Google Search Console:**
   - أضف ملكية `www.qamhad.com` (سجل TXT على مستوى الدومين يغطي الكل).
   - استخدم أداة **Change of Address** من ملكية live إلى www — أهم خطوة لنقل الترتيب رسمياً.
   - أرسل الخرائط الست في ملكية www وأعد فحص صفحات SportsEvent بعد النشر (اختبار Rich Results).
2. **كرون:** فعّل `*/1-5 دقائق` لـ `/cron/ping` و`/cron/notify` على الاستضافة.
3. **الروابط الخلفية:** حدّث ما تملكه من روابط (حسابات التواصل، أدلة، مواقع صديقة) لتشير إلى www مباشرة.
4. **بناء سلطة (Off-page):** الكلمة المستهدفة يتصدرها مواقع بسلطة عالية (يلا شوت وأشباهه). المحتوى والتقنية أصبحا جاهزين؛ يتبقى بناء روابط خلفية عربية رياضية + نشر أخبار أصلية بوتيرة يومية لدخول Google News/Discover بقوة.
5. **Core Web Vitals:** البنية خفيفة أصلاً؛ راقب تقرير التجربة في GSC بعد النشر (خصوصاً LCP لصور الأخبار 1200px — الخدمة عبر بروكسي الوسائط تدعم الكاش، ويُنصح بـ CDN أمامي).
6. **ملاحظة تقنية:** Google أوقفت رسمياً endpoint الـ Sitemap Ping منذ 2023 — النداء يبقى غير ضار ومُنفَّذ لطلبك، لكن القنوات الفعلية المعتمدة هي: sitemaps في robots.txt + GSC + IndexNow (لـ Bing/Yandex) + Last-Modified/ETag الصادقة، وكلها مفعّلة.
7. **اختياري لاحقاً:** أرشيف نتائج تاريخي (صفحات مواسم/جولات)، وصفحات هبوط ثابتة لكل ديربي كبير (مثل /الكلاسيكو) تتحدث تلقائياً بأقرب مباراة.

## رابعاً: توقعات الترتيب لكلمة «مباريات اليوم بث مباشر»

**الوضع التنافسي:** الكلمة من أعلى الكلمات الرياضية العربية تنافسية؛ الصدارة تتطلب تقنية ممتازة (أصبحت متوفرة) + سلطة دومين + نقرات/سلوك مستخدم.

| المرحلة | التوقع الواقعي |
|---|---|
| أسبوع 1–2 بعد النشر + Change of Address | إعادة زحف شاملة؛ انتقال الأرشفة إلى www بدون فقدان (301 + canonical + خرائط). اختفاء أخطاء SportsEvent من GSC خلال أول إعادة فحص |
| شهر 1–2 | ظهور صفحات المباريات الفردية في نتائج «مباراة X وY بث مباشر» و«القنوات الناقلة لمباراة…» — منافسة أقل بكثير من الكلمة العامة وحركة تراكمية كبيرة (Long-tail هو محرك النمو الأساسي) |
| شهر 2–4 | الصفحة الرئيسية/`/matches` تدخل صفحات النتائج 2–3 للكلمة الرئيسية بفضل ItemList + العناوين المطابقة + الإشارات الحية (lastmod/ETag/IndexNow). بدء ظهور الأخبار في Discover إذا انتظم النشر |
| شهر 4–8 | مع نمو الروابط الخلفية والنقرات، دخول الصفحة الأولى (مراكز 5–10) وارد للكلمة الرئيسية؛ الثبات في الثلاثة الأوائل يتطلب سلطة روابط تضاهي المنافسين |

**العوامل الحاسمة بعد النشر:** انتظام الكرون (الإشارات الحية هي الميزة التنافسية هنا)، وتيرة الأخبار، وبناء الروابط. التقنية لم تعد عائقاً — كل عناصر On-page/Technical SEO للكلمة المستهدفة ومشتقاتها الإحدى عشرة مغطاة بنمط قوالب موحد يتوسع تلقائياً لكل مباراة جديدة في JSON.
