# إصلاح خطأ خريطة موقع الفيديو (Google Search Console)

## المشكلة
Search Console كان يعرض الخطأ التالي على `sitemap-video.xml`:

> إن العلامة `<video:content_loc>` أو `<video:player_loc>` لها القيمة `<loc>` نفسها.
> يجب أن تحتوي هذه العلامات على روابط إلى الفيديو الفعلي، وليس إلى الصفحة المقصودة بتنسيق html للفيديو.

السبب: كانت خريطة الفيديو تضع رابط صفحة المشاهدة نفسها (`/video/{id}`) داخل
`<video:player_loc>`، بينما تشترط Google أن تشير هذه العلامة إلى **المشغّل الفعلي**
(رابط embed) أو `<video:content_loc>` إلى **ملف الفيديو** — لا إلى صفحة HTML.

## الحل (3 ملفات)

### 1) `app/Core/VideoFeed.php`
- مخزن دائم جديد لكل فيديو (`player-{id}.json` بجانب مخزن البطاقات) يحفظ:
  - `content_loc` → رابط ملف الفيديو المباشر (mp4 / m3u8) إن وُجد.
  - `player_loc` → رابط المشغّل القابل للتضمين نفسه الذي تعرضه صفحة المشاهدة:
    `https://www.youtube.com/embed/{id}` أو iframe المصدر الخارجي أو مضمّن تغريدة X.
- يُكتب المخزن تلقائياً عند كل استدعاء `find()` (أي عند كل زيارة لصفحة مشاهدة)،
  ودوال جديدة `storedPlayer()` / `playerFor()` للقراءة.

### 2) `app/Controllers/Sitemap.php`
- `videos()` تقرأ الروابط الحقيقية من المخزن، ومع كل بناء للخريطة (كل 15 دقيقة)
  تجلب تفاصيل حتى 15 فيديو غير معروف — فيمتلئ المخزن تدريجياً (180 فيديو ≈ 3 ساعات)
  دون إغراق المصدر بالطلبات.
- الفيديو المعروف الرابط → يُنشر بعلامة `<video:content_loc>` و/أو `<video:player_loc>`
  الصحيحة (مختلفة عن `<loc>` دائماً).
- الفيديو غير المعروف بعد → يُنشر كرابط صفحة عادي **بدون** كتلة `<video:video>`
  (صحيح تماماً في الخريطة ولا يسبب أي خطأ)، ويكتسب كتلة الفيديو تلقائياً لاحقاً.
- تم رفع مفتاح كاش الخريطة إلى `v3` حتى تُبنى النسخة الجديدة فوراً.

### 3) `app/Controllers/Videos.php`
- نفس الخلل كان موجوداً في البيانات المنظمة (JSON-LD `VideoObject`) لصفحة المشاهدة:
  `embedUrl` كان يشير إلى الصفحة نفسها. الآن يشير إلى المشغّل الفعلي
  (يوتيوب embed / iframe خارجي / مضمّن X)، و`contentUrl` إلى ملف/رابط الفيديو.

## التركيب
انسخ الملفات الثلاثة إلى نفس مساراتها على الاستضافة، أو ارفع النسخة الكاملة
`www.qamhad.comFULLv16.zip`. لا حاجة لأي خطوة إضافية — الكاش القديم يتجاوز تلقائياً.

بعد الرفع: في Search Console أعد إرسال `sitemap-video.xml`. الخطأ سيختفي،
وعدد «الفيديوهات التي تم اكتشافها» سيرتفع تدريجياً خلال الساعات الأولى
كلما امتلأ مخزن روابط المشغّل.
