# Btolat Videos JSON API — PHP

**المؤلف:** Manus AI  
**الإصدار:** 1.0.0

هذا مشروع PHP خفيف بلا مكتبات خارجية يحوّل صفحات فيديوهات [بطولات](https://mobile.btolat.com/videos) إلى واجهة JSON منظمة. يستخرج **العنوان، ورابط صفحة الفيديو، والصورة المصغّرة، والتصنيف، وتاريخ النشر عند توفره**، ويدعم قسم «الكل» وصفحات البطولات والفرق والتنقّل بين الصفحات.

> **مهم:** المشروع يقرأ محتوى عامًا من الموقع المصدر ولا يتجاوز تسجيل الدخول أو الحماية. بنية الموقع قد تتغير مستقبلًا، وعندها قد يلزم تحديث محددات HTML. احترم شروط استخدام المصدر وحقوق المحتوى، واستخدم التخزين المؤقت لتجنب الطلبات المفرطة.

## المتطلبات

| المتطلب | القيمة |
|---|---|
| PHP | 8.1 أو أحدث |
| الإضافات | `curl`, `dom`, `libxml`, `json`, `mbstring` |
| خادم الويب | Apache أو Nginx أو خادم PHP المدمج |
| صلاحية الكتابة | على مجلد `storage/cache` |

## التشغيل السريع

من داخل مجلد المشروع شغّل:

```bash
php -S 0.0.0.0:8080 -t public
```

ثم افتح:

```text
http://localhost:8080/?endpoint=videos&category=all&page=1
```

يمكن كذلك رفع المجلد كاملًا إلى استضافة مشتركة واستخدام `index.php` الموجود في الجذر. ملف `.htaccess` يضيف مسارات مريحة عند تفعيل `mod_rewrite`.

## نقاط الواجهة

| الطلب | النتيجة |
|---|---|
| `/?endpoint=videos&category=all&page=1` | أول دفعة من جميع الفيديوهات |
| `/?endpoint=videos&category=world-cup&page=1` | فيديوهات كأس العالم |
| `/?endpoint=videos&category=al-ahly&page=2` | الصفحة الثانية من فيديوهات الأهلي |
| `/?endpoint=videos&category=all&page=1&pages=10` | جمع أول عشر دفعات في JSON واحد |
| `/?endpoint=videos&category=premier-league-en&page=1&pages=3` | جمع ثلاث صفحات من الدوري الإنجليزي |
| `/?endpoint=videos&category=all&page=1&enrich=1` | محاولة إضافة رابط المشغّل لكل فيديو في الصفحة |
| `/?endpoint=video&id=93993` | تفاصيل فيديو واحد وبيانات المشغّل المتاحة |
| `/?endpoint=categories` | كل مفاتيح التصنيفات الجاهزة |
| `/?endpoint=health` | فحص عمل الخدمة |

عند استخدام Apache يمكن استعمال `/videos`, و`/categories`, و`/video/93993`, و`/health` بدل معاملات `endpoint`.

## معاملات الفيديوهات

| المعلمة | الافتراضي | القيم | الشرح |
|---|---:|---|---|
| `category` | `all` | مفتاح من `/categories` | البطولة أو الفريق |
| `page` | `1` | من 1 إلى 100 | أول صفحة مطلوبة |
| `pages` | `1` | من 1 إلى 10 | عدد الصفحات المطلوب دمجها |
| `enrich` | `0` | `0` أو `1` | فحص صفحة كل فيديو لاكتشاف المشغّل؛ يُسمح مع صفحة واحدة فقط |
| `refresh` | `0` | `0` أو `1` | تجاهل التخزين المؤقت |

قسم `all` في الموقع لا يستخدم أرقام صفحات ثابتة؛ بل يستخدم مؤشّر آخر فيديو وتاريخه. يحاكي المشروع هذه الآلية داخليًا، ولذلك يستطيع الوصول إلى الدفعات التالية عبر `page=2` إلى `page=100`. أما البطولات والفرق فتُجلب مباشرة باستخدام أرقام صفحات المصدر.

## روابط التشغيل

`page_url` هو رابط صفحة الفيديو المؤكد. بعض فيديوهات المصدر تعمل عبر X أو YouTube أو مزوّد آخر ولا تملك رابط MP4 مباشرًا داخل HTML. عند استخدام `endpoint=video` أو `enrich=1` يعيد المشروع الحقول التالية عندما يكتشفها:

| الحقل | الدلالة |
|---|---|
| `provider` | `x`, أو `youtube`, أو `direct` |
| `embed_url` | رابط يمكن استخدامه للتضمين أو رابط منشور الفيديو |
| `external_url` | رابط الفيديو عند مزوّد خارجي |
| `media_url` | رابط ملف مباشر عندما يكون موجودًا فعلًا |

القيمة `null` تعني أن الرابط غير ظاهر في HTML العام، ولا تعني وجود خطأ في بطاقة الفيديو.

## التصنيفات الجاهزة

تشمل القائمة الدوري المصري، وكأس الأمم الإفريقية، ودوري أبطال أفريقيا، والدوري السعودي، ودوري أبطال أوروبا، والدوري الإنجليزي، والدوري الإسباني، والدوري الأوروبي، والدوري الإيطالي، والدوري الألماني، والدوري الفرنسي، وكأس العالم، ومنتخب مصر، والأهلي، والزمالك، وريال مدريد، وبرشلونة، وليفربول، ومانشستر سيتي، ومانشستر يونايتد، وأرسنال، ويوفنتوس، وبيراميدز، وباريس سان جيرمان. استخدم `/categories` للحصول على المفاتيح الحالية وروابط API الجاهزة.

## الإعداد والنشر

يمكن ضبط البيئة بالقيم التالية:

| المتغير | الافتراضي | الوظيفة |
|---|---:|---|
| `CACHE_DIR` | `storage/cache` | مسار ملفات التخزين المؤقت |
| `CACHE_TTL` | `300` | مدة التخزين بالثواني |
| `CORS_ALLOW_ORIGIN` | `*` | النطاق المسموح له بطلب الواجهة |

في Nginx اجعل `public/` هو `document_root` ومرّر ملفات PHP إلى PHP-FPM. في الإنتاج يُفضّل تعيين `CORS_ALLOW_ORIGIN` إلى نطاق تطبيقك بدل `*`، ومنح مستخدم PHP صلاحية الكتابة على مجلد التخزين فقط.
