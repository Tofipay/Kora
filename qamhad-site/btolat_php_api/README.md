# Btolat Videos JSON API (v1.0.1)

هذا المشروع عبارة عن واجهة برمجة تطبيقات (API) احترافية مبنية باستخدام PHP، مصممة لاستخراج بيانات الفيديوهات من موقع Btolat.com وتحويلها إلى صيغة JSON سهلة الاستخدام. يدعم المشروع استخراج الفيديوهات من قسم "الكل"، ومن تصنيفات البطولات والفرق المختلفة، مع دعم كامل للترقيم (pagination) واستخراج تفاصيل الفيديو بما في ذلك روابط المشغل المضمنة.

## الميزات

*   **استخراج شامل:** يجلب العنوان، الرابط، الصورة المصغرة، التصنيف، وتاريخ النشر لكل فيديو.
*   **دعم التصنيفات:** يمكن استخراج الفيديوهات حسب تصنيفات محددة مثل "كأس العالم"، "الدوري المصري"، "الأهلي"، وغيرها.
*   **ترقيم الصفحات:** يدعم جلب الفيديوهات عبر صفحات متعددة، بما في ذلك إمكانية جلب ما يصل إلى 10 صفحات في طلب واحد.
*   **تفاصيل الفيديو:** يوفر نقطة نهاية مخصصة لجلب تفاصيل فيديو فردي، بما في ذلك محاولة اكتشاف روابط المشغل المباشرة أو روابط التضمين (embed URLs) من مصادر مثل YouTube، X (Twitter)، و ld+json.
*   **تخزين مؤقت ذكي:** يستخدم نظام تخزين مؤقت لتقليل عدد الطلبات إلى الموقع المصدر وتسريع الاستجابات.
*   **مخرجات JSON احترافية:** يعرض البيانات بتنسيق JSON واضح ومنظم.
*   **سهولة النشر:** يتضمن ملف `.htaccess` لدعم التشغيل على خوادم Apache مع مسارات نظيفة.

## التثبيت والتشغيل

1.  **فك ضغط المشروع:** قم بفك ضغط ملف `btolat_php_api_v1.0.1.zip` في مجلد على خادم الويب الخاص بك.
2.  **إعداد مجلد التخزين المؤقت:** تأكد من أن مجلد `storage/cache` قابل للكتابة بواسطة خادم الويب.

### التشغيل باستخدام خادم PHP المدمج (للتطوير والاختبار)

انتقل إلى مجلد المشروع ونفّذ الأمر التالي:

```bash
cd /path/to/btolat_php_api
php -S 0.0.0.0:8080 -t public
```

سيقوم هذا بتشغيل خادم ويب على المنفذ 8080. يمكنك بعد ذلك الوصول إلى الواجهة عبر `http://localhost:8080/` أو `http://your_server_ip:8080/`.

### التشغيل على خادم ويب (Apache/Nginx)

*   **Apache:** إذا كنت تستخدم Apache، فتأكد من تمكين `mod_rewrite` وأن ملف `.htaccess` في المجلد الجذر للمشروع يعمل بشكل صحيح.
*   **Nginx:** ستحتاج إلى إعداد `location` block في ملف تهيئة Nginx الخاص بك لتوجيه جميع الطلبات إلى `public/index.php`.

## استخدام الواجهة

### 1. فحص حالة الخدمة

`GET /?endpoint=health`

**الاستجابة:**

```json
{
    "success": true,
    "message": "API is healthy"
}
```

### 2. جلب قائمة التصنيفات المدعومة

`GET /?endpoint=categories`

**الاستجابة:**

```json
{
    "success": true,
    "meta": {
        "count": 25
    },
    "data": [
        {
            "name": "الكل",
            "key": "all",
            "type": "all",
            "path": "/videos"
        },
        {
            "name": "الدوري المصري",
            "key": "egyptian-league",
            "type": "league",
            "id": 1000,
            "slug": "egyptian-league",
            "path": "/league/1000/egyptian-league/videos"
        },
        // ... المزيد من التصنيفات
    ]
}
```

### 3. جلب الفيديوهات

`GET /?endpoint=videos&category={category_key}&page={page_number}&pages={number_of_pages}`

**المعلمات:**

*   `category_key` (مطلوب): مفتاح التصنيف. يمكن الحصول عليه من نقطة النهاية `categories`. استخدم `all` لجلب جميع الفيديوهات.
*   `page` (اختياري، الافتراضي: 1): رقم الصفحة المراد جلبها.
*   `pages` (اختياري، الافتراضي: 1): عدد الصفحات المتتالية المراد جلبها بدءًا من `page`. الحد الأقصى هو 10 صفحات في الطلب الواحد.
*   `enrich` (اختياري، الافتراضي: false): إذا كانت `true`، فسيحاول جلب تفاصيل المشغل لكل فيديو في القائمة. قد يؤدي هذا إلى إبطاء الاستجابة بشكل كبير.

**مثال:** جلب أول صفحتين من فيديوهات "كأس العالم":

`GET /?endpoint=videos&category=world-cup&page=1&pages=2`

**الاستجابة:**

```json
    ]
}
```

### 4. جلب تفاصيل فيديو فردي

`GET /?endpoint=video&id={video_id}`

**المعلمات:**

*   `id` (مطلوب): معرّف الفيديو. يمكن الحصول عليه من نقطة النهاية `videos`.
