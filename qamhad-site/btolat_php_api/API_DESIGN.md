# تصميم واجهة Btolat Videos JSON API

**المؤلف:** Manus AI

## الهدف

يوفّر المشروع طبقة PHP مستقلة تقرأ صفحات الفيديو العامة في موقع بطولات وتحوّل بطاقات الفيديو إلى JSON موحّد. يعتمد قسم «الكل» على مؤشّر التحميل المتتابع الخاص بالموقع، بينما تستخدم صفحات البطولات والفرق معامل الصفحة `p`.

## نقاط الاستخدام

| الطلب | الوظيفة |
|---|---|
| `index.php?endpoint=videos&category=all&page=1` | أحدث فيديوهات قسم الكل |
| `index.php?endpoint=videos&category=world-cup&page=1` | فيديوهات كأس العالم |
| `index.php?endpoint=videos&category=premier-league-en&page=2` | الصفحة الثانية من الدوري الإنجليزي |
| `index.php?endpoint=videos&category=all&page=1&pages=10` | جمع عشر دفعات متتابعة بدءًا من الصفحة الأولى |
| `index.php?endpoint=categories` | قائمة التصنيفات المدعومة ومعرّفاتها وروابطها |
| `index.php?endpoint=video&id=93993` | تفاصيل فيديو مفرد ومحاولة اكتشاف رابط التضمين أو المصدر |

## المعلمات

| المعلمة | النوع | القيد | الوصف |
|---|---:|---:|---|
| `endpoint` | نص | `videos`, `categories`, `video`, `health` | العملية المطلوبة |
| `category` | نص | مفتاح من قائمة التصنيفات | الافتراضي `all` |
| `page` | عدد صحيح | من 1 إلى 100 | أول صفحة أو دفعة مطلوبة |
| `pages` | عدد صحيح | من 1 إلى 10 | عدد الصفحات المجمّعة في استجابة واحدة |
| `id` | عدد صحيح | موجب | معرّف الفيديو لنقطة `video` |
| `enrich` | منطقي | `0` أو `1` | جلب تفاصيل التشغيل لكل عنصر؛ مقيّد بعدد صغير لتقليل الطلبات |
| `refresh` | منطقي | `0` أو `1` | تجاهل النسخة المؤقتة عند الحاجة |

## بنية الاستجابة

تحتوي الاستجابة الناجحة على `success`, و`meta`, و`data`. يحتوي كل فيديو على `id`, و`title`, و`page_url`, و`thumbnail`, و`published_at` إن أمكن، وبيانات `category`. عند تفعيل الإثراء أو طلب فيديو مفرد تُضاف `provider`, و`embed_url`, و`media_url` عندما تكون متاحة.

    "page": 1,

## ضوابط الاعتمادية

يقيّد المستخرج الاتصالات بنطاقات بطولات الموثوقة، ويستخدم مهلات اتصال وقراءة، وترويسات HTTP واضحة، ومحاولات محدودة، وتخزينًا مؤقتًا على القرص. تُعاد الأخطاء بصيغة JSON وبحالة HTTP مناسبة. لا يفترض المشروع أن كل بطاقة تملك ملف MP4 مباشرًا؛ إذ قد تكون صفحة الفيديو تضمينًا من X أو YouTube أو مصدر طرف ثالث، ولذلك يبقى `page_url` هو الرابط المؤكد وتُعاد حقول التشغيل عند اكتشافها فقط.
